Microsoft|Security Architecture
Security Boundaries
Data Engineering›DP-700›Security Roles
Security Architecture
Workspace Roles vs OneLake Security
Understanding management boundaries vs granular data access controls in Microsoft Fabric.
🔐 Workspace Roles (Item Management)
👑
Admin / Member / Contributor
Full read/write access to ALL items & data in workspace
👁
Viewer Role
Read-only access; governed by OneLake Data Access Roles
🛠
Scope
Item creation, pipeline execution, workspace settings
🔓 OneLake Data Access Roles (Granular Security)
🗃
Folder & Table Security
Grant access to specific Lakehouse tables or file folders
👤
Applies To
Users with Viewer workspace role or direct Item Shares
⚠
Bypass Rule
Admin/Member/Contributor roles BYPASS OneLake security
💡
Critical Exam Fact
OneLake Data Access Roles ONLY apply to users with the Viewer role. Admin, Member, and Contributor roles have full elevated access and bypass table/folder security restrictions!

Microsoft|SQL Security
RLS & OLS Security
Data Engineering›DP-700›RLS / OLS
Fine-Grained Security
Row-Level & Column-Level Security
Protecting sensitive rows and columns in Fabric Data Warehouse & SQL Analytics Endpoints.
👁
Row-Level Security (RLS)
Restrict row access based on user Microsoft Entra ID identity using T-SQL Security Policies and Inline Table-Valued Functions.
💳
Column-Level Security (CLS)
Restrict access to specific sensitive columns (e.g. Salary, PII, SSN) using T-SQL GRANT SELECT ON Table(Column).
💻 T-SQL Example: Row-Level Security Predicate Function
CREATE FUNCTION Security.fn_securitypredicate(@Region AS sysname) RETURNS TABLE WITH SCHEMABINDING AS RETURN SELECT 1 AS fn_securitypredicate_result WHERE @Region = USER_NAME() OR USER_NAME() = 'admin@company.com';

Microsoft|Data Masking
Dynamic Data Masking
Data Engineering›DP-700›DDM Protection
PII Protection
Dynamic Data Masking (DDM)
Obfuscate sensitive PII data on-the-fly without altering actual storage on OneLake Delta tables.
🔒
default()
Replaces data with zero or 'xxxx' depending on column data type.
📧
email()
Exposes first letter and domain (`aXXX@XXXX.com`).
💳
partial()
Custom prefix and suffix masking for Credit Cards / SSNs.
🎲
random()
Masks numeric fields with random values within range.
💻 T-SQL Statement to Mask Column
ALTER TABLE Sales.Customers ALTER COLUMN EmailAddress ADD MASKED WITH (FUNCTION = 'email()');

Microsoft|Shortcut Security
Multi-Cloud Governance
Data Engineering›DP-700›Shortcut Security
Cross-Cloud Security
Shortcut Security & Credentials
How OneLake secures virtual shortcuts to external clouds (ADLS, AWS S3, GCS).
🔑
Delegated User Identity
Shortcuts use organizational cloud credentials (e.g. AWS IAM Role, Azure Service Principal) to access source data.
🛡
Double Security Boundary
Users must pass BOTH OneLake security AND source storage permissions (e.g. S3 Bucket policy) to read data.
