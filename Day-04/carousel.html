Microsoft|Exam Traps
Exam Trap Alert
Data Engineering›DP-700›Scheduling Trap
Exam Trap #1
The #1 Notebook Scheduling Trap
A critical distinction between standalone notebooks and pipeline orchestration.
❌
The Common Trap
"You can configure a native cron schedule directly inside a PySpark Notebook item settings."
✅
The Correct DP-700 Answer
PySpark Notebooks cannot self-schedule directly. You MUST wrap the notebook in a Data Factory Pipeline or an Airflow DAG to trigger automated runs!
⚠
Why This Matters on the Exam
Multiple-choice questions will offer "Schedule the notebook item directly" as a distractor option. Always eliminate this option immediately! Choose Data Pipelines or Airflow.

Microsoft|Airflow Python Code
Python DAG Syntax
Data Engineering›DP-700›DAG Definition
Code-First Workflow
Fabric Airflow DAG Definition
Creating Python-based Directed Acyclic Graphs (DAGs) to orchestrate Fabric pipelines and notebooks.
💻 Example: Airflow DAG Orchestrating Fabric Notebook Activity
from airflow import DAG from airflow.operators.python import PythonOperator from datetime import datetime, timedelta default_args = { 'owner': 'fabric_data_engineer', 'start_date': datetime(2026, 7, 21), 'retries': 2 } with DAG('fabric_daily_etl_dag', default_args=default_args, schedule_interval='@daily') as dag: task_ingest = PythonOperator(task_id='run_pipeline', python_callable=trigger_fabric_pipeline) task_transform = PythonOperator(task_id='run_notebook', python_callable=trigger_fabric_notebook) task_ingest >> task_transform # Set dependency pipeline -> notebook
